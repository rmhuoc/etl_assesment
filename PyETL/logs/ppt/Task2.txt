 Capa 1: Ingesta de Datos (Data Ingestion Layer)
📌 ¿Qué es?

Es la primera capa del flujo de datos. Se encarga de capturar, extraer y mover los datos desde múltiples fuentes hacia el entorno central de la plataforma (por ejemplo, el Data Lake en ADLS Gen2).
🔄 Funciones principales:

    Conectarse a las fuentes de datos (internas o externas):

        Bases de datos transaccionales (SQL Server, Oracle…)

        APIs (CRM, ERP, herramientas online)

        Archivos (CSV, Excel, JSON, XML)

        Streams en tiempo real (Event Hubs, Kafka)

        Web scraping o sensores IoT (si aplica)

    Extraer datos sin transformarlos (generalmente en formato bruto/raw)

    Cargar datos en el Data Lake (ADLS Gen2) en zonas como:

        Raw zone: datos tal como vienen

        Landing zone: staging temporal

    Registrar metadatos (quién, cuándo, de dónde)

    Validar esquema y calidad mínima (opcional en esta capa)

⚙️ Herramientas típicas en Azure:

    Synapse Pipelines o Azure Data Factory (para orquestación)

    Event Hub / IoT Hub (si hay ingesta en tiempo real)

    Azure Functions / Logic Apps (para automatización)

    Data Management Connector (gobernanza y acceso controlado)

🎯 Objetivo de esta capa:

    Asegurar que todos los datos relevantes llegan de forma segura, repetible y trazable a la plataforma analítica, sin alterar su estructura original.
